❓ ЧАСТЫЕ ВОПРОСЫ (FAQ) ▶
Сохраняются ли мои данные? ▶
Да! Блокнот и календарь сохраняют данные в localStorage браузера — они не пропадут при закрытии страницы. Рецепты и избранное хранятся в базе данных на сервере.
Как масштабировать рецепт на больше людей? ▶
Открой страницу рецепта и измени количество порций в специальном поле. Все ингредиенты пересчитаются автоматически пропорционально.
Можно ли добавить свой рецепт? ▶
Да! Перейди во вкладку "Рецепты" и нажми кнопку "+ СОЗДАТЬ РЕЦЕПТ". Заполни форму с названием, ингредиентами и шагами приготовления.
Почему не загружаются рецепты? ▶
Скорее всего бэкенд не запущен. Убедись что Docker запущен командой "docker compose up" из корневой папки проекта.
Как удалить блюдо из календаря? ▶
Нажми кнопку ✕ справа от блюда в ячейке дня. Блюдо сразу удалится.
Как убрать рецепт из избранного? ▶
Открой вкладку "Избранное" и нажми кнопку ★ справа от рецепта. Или нажми ★ на карточке в ленте рецептов — иконка снова станет ☆.
Работает ли сайт на телефоне? ▶
Да, сайт адаптирован для мобильных устройств. Все страницы корректно отображаются на экранах разного размера.
⚡ API ЭНДПОИНТЫ ▶
Бэкенд работает на порту :8080. Все запросы к рецептам через /api/recipes.
МЕТОД ENDPOINT ОПИСАНИЕ
GET /health Проверка работы сервера
GET /api/recipes Получить все рецепты
GET /api/recipes?search=паста Поиск рецептов по названию
GET /api/recipes/:id Получить рецепт по ID
POST /api/recipes Создать новый рецепт
DELETE /api/recipes/:id Удалить рецепт по ID
GET /api/tags Получить все теги
GET /api/categories Получить все категории
POST /api/recipes/:id/favorite Добавить рецепт в избранное
GET /api/shopping-list Получить список покупок
⚙️ ТЕХНОЛОГИИ ПРОЕКТА ▶
🎨
ФРОНТЕНД
HTML5, CSS3, JavaScript — без фреймворков. Шрифты: Press Start 2P (заголовки), Nunito (текст).
⚙️
БЭКЕНД
Go 1.21 + Gin framework. REST API с CORS поддержкой. Порт: 8080.
🗄️
БАЗА ДАННЫХ
SQLite. Таблицы: Users, Recipes, Ingredients, Recipe_Steps, Categories, Meal_Plans.
🐳
ДЕПЛОЙ
Docker + Docker Compose. Фронтенд через Nginx, бэкенд в отдельном контейнере.
